## О проекте

Spring Boot-приложение, которое получает сведения о товарах из внешнего сервиса “Fake Store API” (https://fakestoreapi.com),
сохраняя их в базу данных, а затем позволяя работать с этой информацией через собственное REST API (создание, просмотр, 
редактирование и удаление). После первоначального импорта все операции с товарами выполняются только внутри этого приложения, 
без повторного обращения к внешнему сервису.

### Подробнее о реализованных операциях

1.	Создан эндпоинт импорта товаров из внешнего API (https://fakestoreapi.com/products) с сохранением всех полей в базу данных. Использованы JPA-сущности: Product, Category, Rating.
2.	Реализованы базовые HTTP-операции (создание, чтение, обновление, удаление) для работы с товарами внутри приложения (локально, без взаимодействия с внешним API), а именно:
   -	создание нового товара;
   -	получение товара по ID;
   -	получение списка всех товаров (поддержка пагинации: page, size);
   -	редактирование существующего товара;
   -	удаление товара.
3.	Добавлена фильтрация по стоимости (параметры minPrice и maxPrice в формате десятичной дроби, два числа после запятой указываются опционально, т.е. метод принимает “15” и “15.75”).
4.	Создан метод получения списка уникальных категорий товаров.
5.	Добавлен метод получения товаров по категории.
6.	Реализован повторный импорт: если товар с таким id уже существует, он обновляется.
7.	Добавлена возможность сортировки товаров по любому из полей и сразу по несколькоим полям с указанием отдельного направления для каждого из этих полей (возрастание/убывание).
8.	Реализован планировщик (@Scheduled), синхронизирующий товары из внешнего API каждые 30 минут.
   В случае локальных правок товаров в БД или их удаления, приоритет при импорте отдаётся внешней системе (добавление недостающих, перезапись измененных).
9.	Подключена базовая авторизация на основе Spring Security и сущности User в БД.
10.	Реализованный функционал покрыт unit-тестами.
11.	Эндпоинты описаны через Swagger UI.



## Перед тем, как запустить

Необходимо установить PostgreSQL 17 и создать БД с именем FakeStore
